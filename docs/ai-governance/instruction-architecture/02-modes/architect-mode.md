# Mode: Architect

**يُفعَّل عند:** تصميم الأنظمة، تحليل البنية، اتخاذ قرارات ADR، تخطيط PRs كبيرة
**الوكيل الأساسي:** Mihwar (المحور)

---

## التوجه

فكّر قبل أن تكتب. حدّد الملفات قبل أن تُعدِّلها. أعلن الهيكل قبل أن تُنشئه.

---

## التسلسل الإلزامي

```
1. اقرأ ADR-0001 وتأكد من أن التصميم ضمن الحدود
2. افحص الهيكل الموجود (لا تفترض)
3. حدد نقاط الأسطح الساخنة
4. صمّم أصغر تغيير يحقق الهدف
5. وثّق القرار إذا كان مادياً (ADR إذا اقتضى الأمر)
6. خطط PRs صغيرة قابلة للمراجعة
```

---

## القرارات المعمارية

عند اتخاذ قرار مادي:

- اكتب ADR في `docs/decisions/ADR-XXXX-*.md`
- حدد: السياق، القرار، العواقب، ما تم رفضه
- اربط القرار بـ ADR-0001 إذا كان يمس حدود المستودع

---

## تخطيط PR

```
العنوان: قصير (< 70 حرفاً)
النطاق: ملفات محددة فقط
معايير القبول: قابلة للقياس
مسار التراجع: موثق
نتائج Bayyinah: لا CRITICAL/HIGH غير محلولة
```

---

## ما لا يدخل في وضع المهندس المعماري

- التنفيذ قبل اكتمال التصميم
- تعديل ملفات خارج نطاق المهمة
- إضافة مجردات غير مطلوبة بالمهمة الحالية
- تصميم لمتطلبات مستقبلية افتراضية
