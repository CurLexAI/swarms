## الرؤية والرسالة

**الرؤية:**  
تمكين المؤسسات من امتلاك سيادة كاملة على أنظمة الذكاء الاصطناعي الوكيلية (Agentic AI)، دون تنازل عن الأمان أو الخصوصية.

**الرسالة:**  
بناء منصة Swarms مفتوحة المصدر (Core) مع امتدادات مؤسسية (Enterprise) تتيح للعملاء تشغيل وكلاء ذكاء اصطناعي محلياً (Ollama) أو سحابياً خاصاً (Self-Hosted)، مع ضمان الامتثال التنظيمي الكامل (مثل CST/NCA، PDPL).

---

## نسخة المؤسس (Founder Edition)

### 1. المبادئ الحاكمة
- **السيادة أولاً**: النموذج الأساسي (Core) يجب أن يعمل كلياً offline على الأجهزة المحلية.
- **لا أبواب خلفية**: لا يجوز تضمين أي مكالمات خارجية غير موثقة أو تبعيات تُسرب البيانات.
- **الحق في التفرع (Fork)**: المستودع العام يحمل ترخيص MIT، ويحق لأي طرف تفرعه والتعديل عليه.

### 2. ما تملكه المؤسس
- **العلامة التجارية:** CurLexAI، وشعار المشروع.
- **البنية التحتية المرجعية:** إعدادات النشر المرجعية على Modal/Google Cloud المُدارة من الفريق المؤسس.
- **المستودع الرئيسي:** `github.com/moteb1989/LexPrim` هو المصدر الوحيد للحقيقة (Source of Truth).

### 3. التزامات المؤسس
- صيانة النواة المفتوحة (Core) باختبارات شاملة وتوثيق كامل.
- إصدار تحديثات أمنية دورية.
- عدم احتكار التحسينات: أي تحسين يُساهم به المؤسس في النواة يجب أن يُعاد إلى المستودع العام.

### 4. صلاحيات المؤسس
- قبول أو رفض المساهمات الخارجية بناءً على الالتزام بالأسس التقنية.
- إصدار نسخ مؤسسية (Enterprise) بميزات إضافية (مثل دعم eSIM، واجهات تحكم متقدمة) بشروط تجارية منفصلة.

## نسخة العملاء (Client Edition)

### 1. حقوق العميل
- **الوصول الكامل للمصدر**: يحق لأي عميل الحصول على كود المصدر الكامل للنسخة التي يستخدمها (بما فيها الإضافات المؤسسية إذا تعاقد عليها).
- **التشغيل الذاتي الكامل**: يُمكن تشغيل المنصة دون أي اتصال خارجي بعد التثبيت.
- **التخصيص**: يُسمح للعميل بتعديل أي جزء من النظام ليتوافق مع بيئته، طالما لا ينتهك التراخيص أو العلامات التجارية.

### 2. نموذجا الترخيص للعميل
| النوع | الوصف |
|-------|-------|
| **النسخة المجتمعية (Community)** | ترخيص MIT، مجانية للأبد. تشمل النواة الأساسية. |
| **النسخة المؤسسية (Enterprise)** | اشتراك سنوي، تشمل دعم eSIM، شبكات 5G خاصة، تكامل MDM، وضمان الامتثال. |

### 3. مستوى الدعم
- المجتمع: Issues على GitHub، منتدى نقاش، وثائق.
- المؤسسات: SLA، قنوات دعم خاصة، تحديثات أمنية عاجلة، استشارات معمارية.

### 4. حماية بيانات العميل
- لا يجمع النظام أي بيانات telemetry دون موافقة صريحة.
- جميع مفاتيح API وشهادات الأمان بحوزة العميل فقط.
- يحق للعميل تدقيق الكود أمنياً (Security Audit) عبر طرف ثالث.

## الأسس التقنية الملزمة للجميع

### 1. العمارة (Architecture)
- **Clean Architecture + Hexagonal (Ports & Adapters)**
- طبقات مفصولة: `Presentation → Application → Domain → Infrastructure`
- كل مكون يُحقَّق عبر Interface، ويُستبدل عبر Dependency Injection.

### 2. معايير الكود (All Contributions)
- **Python 3.11+** مع Type Hints كاملة.
- **تنسيق:** Black + Ruff (line-length 100).
- **فحص ثابت:** mypy (strict mode).
- **Docstrings:** Google Style لكل دالة وكلاس.
- **Header موحد لكل ملف:**

```
Copyright (c) 2025 CurLexAI. Licensed under MIT.

Purpose: <Brief purpose statement>
```

### 3. الاختبارات
- تغطية لا تقل عن 80% للمنطق الحيوي.
- **Unit Tests** بـ `pytest` مع fixtures و mocks.
- **Integration Tests** لـ Ollama API و GitHub API.
- **Security Tests** للمدخلات غير الموثوقة.

### 4. إدارة الأسرار
- لا تُخزَّن أسرار (Tokens, Keys) في الـ repo.
- استخدام `.env` + `Secret Manager` (محلياً أو على Google Cloud).
- توفير `.env.example` مع شرح كل متغير.

### 5. النماذج والوكلاء
- **النماذج المدعومة:** Qwen2.5 (عام/برمجة)، DeepSeek-R1 (استدلال/أمن)، Llama 3.1 (احتياطي).
- **الوكلاء:** يبنون بنمط Strategy أو Chain of Responsibility.
- **التسجيل:** Audit logging لكل استدعاء نموذج (محلي/سحابي).

## الحوكمة وعملية التطوير

### 1. إدارة التغييرات
- **RFC (Request for Comments)** لأي تغيير معماري كبير.
- **Pull Requests** تُراجع من قِبل مشرف واحد على الأقل (للمجتمع) أو اثنين (للمؤسسات).
- **الدمج** يتطلب اجتياز CI (lint, test, security scan).

### 2. الإصدارات (Versioning)
- نتبع **Semantic Versioning**: `MAJOR.MINOR.PATCH`
- أي تغيير يكسر التوافق → MAJOR.
- إضافة ميزة متوافقة → MINOR.
- إصلاح خلل → PATCH.

### 3. دورة الإصدار
- **Edge:** أحدث main، غير مستقر.
- **Beta:** كل أسبوعين، للمختبرين.
- **Stable:** كل شهر، للإنتاج.

## سيادة البيانات والأمان

### 1. الخصوصية حسب التصميم (Privacy by Design)
- لا توجد اتصالات خارجية غير مصرح بها.
- أي استدعاء سحابي (Modal, Vertex AI) يكون اختيارياً ومعزولاً بـ mTLS.
- السجلات (logs) لا تحتوي على بيانات شخصية.

### 2. الامتثال التنظيمي
- **السعودية:** CST/NCA (ECC, CSCC, PDPL).
- **عالمياً:** GDPR-ready (محلياً لا تُرسل بيانات المستخدمين للخارج).

### 3. الاستجابة للثغرات
- الإبلاغ عبر `security@curlex.ai`
- إصلاح خلال 72 ساعة للإصدارات المدعومة.
- إشعار العملاء فوراً.
